# Lyle.ai Site Map

## Current Pages

### index.html
Purpose:
- introduce Lyle
- explain what Lyle is
- establish trust
- position the product clearly
- guide users into the app

### chat.html
Purpose:
- serve as the main Lyle chat interface
- allow users to interact with the assistant
- become the core product experience

## Likely Future Pages
Potential future pages may include:
- about.html
- login.html
- signup.html
- settings.html
- pricing.html
- account.html

## User Flow
Primary path:
1. user lands on index.html
2. user understands what Lyle is
3. user moves into chat.html
4. user begins using the product

## Site Structure Rule
Pages should exist because they serve a clear product purpose, not just because they are common website pages.

## Mapping Principle
Keep the site small, clean, and intentional.
