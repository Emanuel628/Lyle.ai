# Lyle.ai Page Content Plan

## index.html

### Header
Purpose:
- show the Lyle.ai brand
- give simple navigation
- make the product feel real and organized

### Hero Section
Purpose:
- explain what Lyle is immediately
- communicate that Lyle is all-purpose
- communicate stronger coding emphasis
- present Lyle as clean, modern, and useful

### Core Messaging Direction
The landing page should communicate that Lyle is:
- an all-purpose AI assistant
- especially strong at coding help
- built for work, ideas, and general help
- trustworthy and reliable

### Feature Section
The feature section should highlight:
- coding assistance
- file upload and analysis
- image upload and analysis
- image generation
- idea generation
- question answering

### Trust Section
The site should communicate:
- reliability
- trustworthiness
- supportive interaction
- serious product intent

### Product Preview Section
This should show:
- a chat preview
- a clean interface
- a believable product experience

### Call to Action
The CTA should direct users toward trying the chat experience.

## chat.html

### Top Bar
Purpose:
- reinforce the Lyle identity
- keep the interface organized
- hold future controls if needed

### Chat Area
Purpose:
- display conversation clearly
- support both general and technical interactions
- feel focused and readable

### Input Area
Purpose:
- make chatting easy
- make uploading feel possible later
- support the main user action clearly

## Content Rule
The site should describe Lyle honestly as a real product direction, not empty hype.
