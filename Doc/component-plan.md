# Lyle.ai Component Plan

## Shared Components
The product should include shared components such as:
- header
- navigation
- footer
- primary button
- secondary button
- section container

## Landing Page Components
The landing page should include:
- hero section
- feature blocks or cards
- trust section
- product preview panel
- call-to-action section

## Chat Page Components
The chat page should include:
- top bar
- message list
- user message bubble
- assistant message bubble
- chat input area
- send button

## Future Components
Later components may include:
- file upload control
- image upload control
- generated image panel
- settings menu
- auth forms

## Component Style Direction
Components should feel:
- clean
- modern
- uncluttered
- readable
- consistent

## Coding-Focused Support
Because Lyle has a strong coding emphasis, components should support:
- readable responses
- structured content
- clear conversation flow

## Simplicity Rule
Do not overload the product with too many component types too early.

## Reuse Rule
Shared components should be reused wherever possible to keep the interface consistent.
