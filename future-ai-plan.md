# Lyle.ai Future AI Plan

## Core AI Direction
Lyle is intended to be an all-purpose AI assistant with a heavy emphasis on coding assistance.

## Long-Term Capability Direction
Lyle should eventually support:
- coding help
- question answering
- idea generation
- file upload and analysis
- image upload and analysis
- image generation

## Version 1 Direction
Version 1 should include:
- frontend
- backend
- authentication
- security
- Redis
- database
- worker support if needed
- a functioning chatbot

## Version 1.5 Direction
By version 1.5, Lyle should have a functioning chat shell.

## Version 2 and Beyond
Later versions can focus on:
- fine-tuning
- refinement
- stronger quality
- deeper capability improvements

## AI Personality Direction
Lyle should adapt to the user's wording, tone, and emotional cues while staying:
- respectful
- useful
- intelligent
- professional
- safe

## Safety Direction
Lyle should not become:
- a BF / GF substitute
- a BFF replacement
- a tool for planning horrible things

## Product Goal
Lyle should be broadly useful while standing out as especially strong in coding and technical help.

## Future Planning Rule
The AI should grow in capability without becoming confusing, bloated, or unsafe.
