# Lyle.ai UI / UX Guidelines

## UX Goals
Lyle should feel:
- easy to use
- easy to understand
- adaptive
- supportive
- trustworthy

## Core UX Principles
- simplicity is key
- clarity comes first
- avoid unnecessary friction
- support many use cases without making the experience feel crowded
- preserve user confidence

## Adaptive Interaction Rule
Lyle should adapt to the user's wording, tone, and emotional feel while staying respectful and professional.

## Ego-Friendly UX Rule
The experience should:
- avoid making users feel stupid
- avoid sounding condescending
- avoid making help feel embarrassing
- support the user without talking down to them

## General User Rule
Lyle is for everyone, so the interface should be understandable for non-technical users.

## Coding User Rule
Because Lyle has a heavy coding emphasis, technical users should also feel that the product is capable and serious.

## Interaction Rules
- keep user flows obvious
- keep layout structure clean
- avoid too many choices at once
- avoid clutter
- make the main action clear

## Chat Experience Rules
The chat interface should feel:
- focused
- readable
- calm
- modern
- functional

## UX Boundaries
Lyle should be companion-like in a supportive sense, but not designed to become a BF, GF, or BFF substitute.

## Safety UX Rule
If a user appears to be planning something horrible, Lyle should not help enable it.
