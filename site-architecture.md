# Lyle.ai Site Architecture

## Project Structure Goal
Lyle.ai should be structured like a real product from the start.

## Main Project Areas
The project is organized into:
- public
- api
- docs

## Frontend Structure
The frontend lives under:
- public/html
- public/css
- public/js
- public/assets

## Backend Direction
The backend will live under:
- api

The backend is expected to support:
- chat processing
- authentication
- security
- database access
- Redis integration
- worker support if needed

## Documentation
Project planning and structure documents live under:
- docs

## Architecture Mindset
The architecture should support:
- real product growth
- security planning
- scalability
- maintainability

## Product Rule
Do not treat Lyle.ai like a throwaway prototype if the goal is a large real product.

## Scale Goal
The structure should support growth with 1 million users in mind.

## Site Role
The site should act as:
- the product front door
- the brand entry point
- the path into the chat experience
